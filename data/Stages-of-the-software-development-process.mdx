# Etapas del proceso de desarrollo de software tradicionales

<Paragraph>Tenemos una serie de etapas</Paragraph>

- Analisis de requerimientos
- Disenio de la solucion
- Desarrollo y evaluacion
- Despliegue
- Mantenimiento y evolucion

<Paragraph>
  Cada paso tiene una entrada y una salida para dar paso a paso de esto
</Paragraph>

# Análisis de requerimientos

- Descubrimiento
- Analisis y documentacion
- Comprension del problema

<Paragraph>
  En esta etapa vamosa construir lo que necesitamos hacer, analisando
  requerimientos, funcionales y no funcionales un vez termina esta etapa
  continuamos con la siguiente
</Paragraph>

# Diseño de la solución

- Comprension del problema
- Abstraccion
- Modelado
- Solucion Detallada

<Paragraph>
  En esta etapa se hace un analisis profundo en donde se plantean posibles
  soluciones atravez de requerimientos y modelos
</Paragraph>

# Desarrollo y evolución

- Solucion Detallada
- Programacion e integracion
- Automatizacion de pruebas
- Artefacto de software

<Paragraph>
  Ya aqui vamos a iniciar el desarrollo de un producto, aqui evaluamos,
  ejecutamos y todo esto va atado a unos requerimientos de aceptacion
</Paragraph>
<Paragraph>
  Tambien vamosa evaluar la infraestructura de todo lo que requiere nuestro
  proyecto para existir y ser desplegado
</Paragraph>

# Despliegue

- Artefacto de software
- infraestructura
- Operaciones
- software disponible

<Paragraph>
  Aqui utilizamos la infraestructura que planteamos para poner todo disponible a
  ser usado
</Paragraph>

# Mantenimiento y evolución

- Software disponible
- Arreglo de errores
- nuevas funcionale
- Sistema deprecado

<Paragraph>
  Aqui ya nos dedicamos a mantener nuestro software dandole soporte y con
  mejoras continuas para hacer mas viable
</Paragraph>
